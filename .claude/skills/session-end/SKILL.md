# Session End Skill

세션 종료 시 현재 세션에서 변경된 코드 내용을 반영하여 README.md를 업데이트합니다.

## 실행 단계

1. **변경 내용 수집**
   - `git log`로 현재 세션에서 작성된 커밋 내역 확인
   - `git diff main`으로 main 브랜치 대비 변경 사항 확인
   - 변경/추가/삭제된 파일 목록과 내용 파악

2. **README.md 업데이트 작성**
   - 기존 README.md 내용을 읽음
   - 현재 세션의 코드 변경 내용을 반영하여 README.md 업데이트
   - **중요: 모든 내용은 한글로 작성**
   - 프로젝트 구조, 기능 설명, 실행 방법 등을 최신 상태로 유지

3. **사용자 리뷰 요청**
   - AskUserQuestion 도구를 사용하여 업데이트된 README.md 내용을 사용자에게 보여주기
   - 옵션 제공:
     - "이대로 진행" - 내용 그대로 커밋 및 푸시
     - "내용 수정" - 사용자가 직접 수정 요청
     - "취소" - 업데이트 취소

4. **커밋 및 푸시**
   - 사용자 승인 후 README.md를 수정하고 저장
   - `/commit` 스킬을 사용하여 커밋
   - 현재 브랜치에 푸시
